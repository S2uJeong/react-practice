# React 연습장

# 개요

### 철학

- Dom 조작에 대한 관리 어려움 → 리액트는 선언적이며 컴포넌트 기반의 접근 방식 제시
- 모든 UI를 컴포넌트로 바로한다.
1. 컴포넌트 기반 아키텍처 
2. 가상 DOM : 변경한 요소만 실제 DOM에 반영하여 렌더링 성능 개선.
3. 선언적 프로그래밍
    - 명령형 : 어떻게 변경할지 작성해야함 (값 수정 + 화면 출력)
    - 선언형 : 무엇을 보여줄지 선언함 (값 수정)

### 특징

- 단방향 데이터 흐름 : 부모 → 자식 , 자식 -x→ 부모
- JSX (JavaScript XML) 사용
    - 문법 확장으로, HTML과 유사한 형태로 UI를 정의할 수 있음
- 상태 관리 도구 제공
    - 훅 : 함수형 컴포넌트에서 상태 관리, 생명주기와 관련한 기능을 사용할 수 있도록 하는 리액트 내장 기능
        
        `useState` , `useReducer` : 로컬 상태 관리를 위한 기본 훅
        
        `useEffect` : 컴포넌트 생명주기 처리를 위한 훅 
        
    - Context API : 전역 상태 관리, 여러 컴포넌트 간 데이터 공유 쉬움
